A lightweight, synthesizable 2-Bit Pulse Width Modulation (PWM) Generator implemented in Verilog. This project demonstrates basic digital design concepts, including sequential counters and combinational logic comparison to generate varying duty cycle square waves.

# Features

Variable Duty Cycle: Supports 4 discrete duty cycle levels (00, 01, 10, and 11) using a 2-bit input selection.
Synchronous Operation: Built around a standard positive-edge triggered clock counter.
Asynchronous Reset: Includes a dedicated master reset signal to clear the internal counter instantly.

# Architecture & Design

The module utilizes an internal 2-bit free-running counter that increments on every clock cycle. The PWM output signal is generated by continuously comparing the counter value against the input duty cycle value (dty).

Condition: If counter < dty, the output (pwm_out) drives HIGH (1'b1).
Condition: Otherwise, the output drives LOW (1'b0).

# Duty Cycle Mapping

Given a 2-bit counter (0 to 3):

* dty = 2'b00 rightarrow$ 0% Duty Cycle (Counter is never less than 0)
* dty = 2'b01 rightarrow$ 25% Duty Cycle (High when counter = 0)
* dty = 2'b10 rightarrow$ 50% Duty Cycle (High when counter = 0, 1)
* dty = 2'b11 rightarrow$ 75% Duty Cycle (High when counter = 0, 1, 2)

#  Module Descriptions

 1. PWM Signal Generator (pwm_signal.v)

The core design module that handles the counter increments and output generation logic.

verilog
module pwm_signal(
    input [1:0] dty,   // 2-bit Duty Cycle selection
    input clk,         // Clock input
    input rst,         // Asynchronous active-high reset
    output pwm_out     // Generated PWM output signal
);

 2. Testbench (pwm_tb.v)

A comprehensive simulation environment that initializes the system, toggles the clock signal, applies reset, and steps through different duty cycle configurations (2'b01 and 2'b11) over a time frame of 550ns before terminating.

# Simulation & Testbench

The testbench sets up a clock with a time period of 2 time units (always #1 clk=~clk;).

# Simulation Flow:

1. 0ns - 2ns: Active-high reset (rst = 1) initializes the internal counter to 0.
2. 2ns: Reset is de-asserted, and the duty cycle input is set to 2'b01 (25% Duty Cycle).
3. 152ns: Duty cycle transitions to 2'b11 (75% Duty Cycle).
4. 552ns: Simulation completes via the $finish system task.
